📋 [CONTEXT]:
Project: BRINKS RouteMaster - a premium PWA for a Brinks (cash logistics) team leader.
Purpose: Daily route optimization for 10-25 stops with time windows, VIP priorities,
traffic-aware ETAs, smart breaks, live navigation mode with geofenced arrival detection,
Waze deep-link navigation, and end-of-day analytics. Hebrew RTL UI, dark premium design.
Tech Stack: React 18 + TypeScript + Vite (PWA) | FastAPI + SQLAlchemy | PostgreSQL (Docker)
| Google OR-Tools | Google Maps Platform | Anthropic Claude API (OCR).
Current Phase: 1 of 5 (Foundation)
Last Task: None - fresh project.
Current Status: Empty repository containing only .cursorrules and docs/spec.md.

───────────────────────────────────────────────────

🔨 [TASK]:
Scaffold the complete project foundation: monorepo structure, database, backend skeleton,
frontend skeleton with the full design system, and JWT authentication. NO business
features yet (no routes/stops logic) - foundation only, but visually polished.

Sub-tasks:
1. Create monorepo structure:
   /backend  - FastAPI app (src/main.py, src/core/config.py, src/models/, src/api/,
               src/services/, src/db/session.py, requirements.txt, .env.example)
   /frontend - Vite React-TS app (src/styles/tokens.css, src/styles/global.css,
               src/components/ui/, src/pages/, src/api/client.ts, src/App.tsx)
   Root: docker-compose.yml (postgres:16 + volumes), project_plan.md, README.md, .gitignore

2. Database (SQLAlchemy models + Alembic migration) - includes the LEARNING ENGINE
   tables from day one (populated in Phase 4, used in Phase 5):
   - users: id, username, password_hash, full_name, role ('team_leader'|'manager'), created_at
   - depots: id, name, address, lat, lng  (single Brinks branch, seeded)
   - customers: id, name, normalized_address (UNIQUE with name), lat, lng,
     category ('bank_branch'|'atm'|'retail_chain'|'private_business'|'other'),
     default_service_min INT NULL, learned_service_min FLOAT NULL,
     learned_service_p80 FLOAT NULL, service_sample_count INT DEFAULT 0,
     parking_lat FLOAT NULL, parking_lng FLOAT NULL, parking_sample_count INT DEFAULT 0,
     geocode_confidence FLOAT NULL, notes, created_at
     (persistent customer registry - the memory of the system. category powers
      cold-start defaults; parking_lat/lng is the LEARNED real stopping point,
      which beats Google's address pin; p80 powers risk-aware ETA for deadlines)
   - routes: id, user_id FK, date, status ('planning'|'active'|'completed'),
     departure_time, break_duration_min, break_window_start TIME, break_window_end TIME,
     deadline_buffer_min INT DEFAULT 10, vip_weight FLOAT DEFAULT 1.0,
     variance_mode BOOL, optimized_at TIMESTAMP NULL,
     naive_duration_min INT NULL, optimized_duration_min INT NULL,
     solver_explanation JSONB NULL, created_at
     (naive vs optimized = the "time saved" sales metric; solver_explanation stores
      why this order was chosen - which stops drove the structure)
   - stops: id, route_id FK, customer_id FK NULL, customer_name, address, lat, lng,
     sequence_order INT, locked BOOL, priority ('normal'|'vip'),
     tw_type ('none'|'before'|'after'|'window'), tw_start TIME NULL, tw_end TIME NULL,
     service_duration_min INT DEFAULT 10, service_estimate_source ('default'|'category'|'learned'),
     notes TEXT, status ('pending'|'arrived'|'done'|'skipped'), eta TIMESTAMP NULL,
     actual_arrival TIMESTAMP NULL, actual_departure TIMESTAMP NULL,
     actual_park_lat FLOAT NULL, actual_park_lng FLOAT NULL,
     exception_code ('none'|'not_ready'|'gate_wait'|'parking'|'closed'|'branch_busy'|'other'),
     exception_note TEXT NULL
   - service_samples: id, customer_id FK, stop_id FK, duration_min FLOAT,
     day_bucket ('weekday'|'friday'|'holiday_eve'), hour_bucket INT (0-23),
     exception_code, is_outlier BOOL DEFAULT false, recorded_at
     (each geofence enter->exit measurement: the real time from stepping off the
      truck to driving away. Samples flagged with an exception are excluded from
      the learned median so one bad day never poisons the estimate)
   - leg_samples: id, from_lat, from_lng, to_lat, to_lng, from_customer_id NULL,
     to_customer_id NULL, hour_bucket INT (0-23),
     day_bucket ('weekday'|'friday'|'holiday_eve'),
     predicted_min FLOAT (Google), actual_min FLOAT, recorded_at
     (real vs predicted travel time per leg per time-of-day - powers the learned
      ETA correction factor)
   - work_days: id, user_id FK, date UNIQUE(user_id,date), route_id FK NULL,
     start_at TIMESTAMP NULL, end_at TIMESTAMP NULL, break_min INT DEFAULT 0,
     driving_min INT DEFAULT 0, service_min INT DEFAULT 0, waiting_min INT DEFAULT 0,
     total_min INT NULL, overtime_min INT DEFAULT 0,
     manually_edited BOOL DEFAULT false, edit_note TEXT NULL, created_at
     (the timesheet: auto-filled from depot geofence exit/entry in Phase 4.
      Any manual correction sets manually_edited=true - the record stays auditable)
   - user_settings: id, user_id FK UNIQUE, standard_day_min INT DEFAULT 516,
     standard_week_min INT DEFAULT 2520, geofence_radius_m INT DEFAULT 150,
     theme ('dark'|'light') DEFAULT 'dark'
     (work-hour standards MUST be configurable - final values come from Brinks
      payroll/collective agreement, not hardcoded)
   - route_events: id, route_id FK, type, payload JSONB, created_at  (audit trail,
     powers the future manager dashboard - design for it now)
   Seed script: one team_leader user + user_settings row + one depot
   ("סניף ברינקס", placeholder coords).
